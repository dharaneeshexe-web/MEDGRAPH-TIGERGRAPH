Evaluation methodology & insights
Token reduction — 64.2%
GraphRAG returns a compact structured entity list (~50 tokens of context) derived from 2-hop graph traversal, versus Basic RAG which injects 5 full abstract chunks (~600 tokens). For co-mention queries the LLM only needs the entity list — full abstract prose is wasted context. Calculated as (648 − 232) / 648 × 100 = 64.2%, averaged over 5 questions. Cost savings calculated at Groq Llama 3.3 70B pricing ($0.59/1M input, $0.79/1M output).
LLM-as-a-judge — 100% PASS (Bonus ≥ 90% achieved)
Judge prompt: Groq Llama 3.3 70B at temperature=0. Format: PASS/FAIL with one-sentence rationale. Grading criteria: factual correctness of named entities, no hallucination of drug/disease names, complete coverage of the question scope. All 15 answers (3 pipelines × 5 questions) received PASS. The ≥ 90% bonus threshold is achieved.
BERTScore F1 — roberta-large raw
BERTScore measures semantic similarity between generated answer and reference answer using contextual embeddings from roberta-large. Raw F1 range for correct answers on this model: 0.83–0.87. GraphRAG scores 0.8367 vs Basic RAG 0.8451 — a 0.84% difference attributable to GraphRAG returning structured entity lists (structurally distinct from prose reference answers) rather than a factual degradation. The LLM judge confirms content equivalence. The raw ≥ 0.88 bonus threshold was not achieved; rescaled BERTScore not computed.
Entity recall
Custom metric: for each question, an expected entity list was compiled (e.g. for "drugs co-mentioned with diabetes": metformin, insulin, pioglitazone, glipizide, sitagliptin). Entity recall = fraction of expected entities present in the generated answer. GraphRAG (0.560) outperforms Basic RAG (0.453) — graph traversal retrieves structurally relevant co-mention entities, while FAISS retrieves text-similar chunks that may not enumerate all relevant entities explicitly. LLM-only scores highest (0.633) due to training data coverage of well-known entity relationships.
