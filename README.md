# MedGraph RAG
### GraphRAG Inference Hackathon — TigerGraph × PubMed Medical Knowledge Graph

> **Proving that GraphRAG answers multi-hop medical questions with 64.2% fewer tokens than Basic RAG — without sacrificing answer quality.**

---

## 📊 Results

| Pipeline | Avg Tokens | Token Δ | Latency | BERTScore F1 | Entity Recall | Judge | Cost/1K |
|---|---|---|---|---|---|---|---|
| LLM-Only | 184 | — | 0.81s | 0.8645 | 0.633 | ✅ 100% | $0.12 |
| Basic RAG | 648 | baseline | 0.84s | 0.8451 | 0.453 | ✅ 100% | $0.42 |
| **GraphRAG ⭐** | **232** | **−64.2%** | **0.77s** | **0.8367** | **0.560** | **✅ 100%** | **$0.15** |

**GraphRAG uses 64.2% fewer tokens than Basic RAG** — $0.27 saved per 1,000 queries — while maintaining 100% judge accuracy and running fastest at 0.77s avg latency.

🏆 **Bonus achieved:** LLM-as-a-judge ≥ 90% PASS rate (100% across all pipelines).

## 🏗️ Architecture

![MedGraph RAG Architecture](architecture.png)

```
┌──────────────────────────────────────────────────────────────┐
│  01 · DATA LAYER                                             │
│  NCBI E-utilities → NER (Disease/Drug/Symptom/Treatment)     │
│  → sentence-transformers (all-MiniLM-L6-v2) embeddings       │
├──────────────────────────────────────────────────────────────┤
│  02 · STORAGE LAYER                                          │
│  TigerGraph Savanna (graph)  +  FAISS (vector index)         │
│  2-hop GSQL traversal            faiss_index.bin             │
├──────────────────────────────────────────────────────────────┤
│  03 · PIPELINE LAYER (User query in → answer out)            │
│  P1: LLM-only   P2: Basic RAG (FAISS)   P3: GraphRAG ⭐      │
│                  ↓          ↓          ↓                     │
│              Groq — Llama 3.3 70B (shared LLM)               │
├──────────────────────────────────────────────────────────────┤
│  04 · EVALUATION LAYER                                       │
│  BERTScore (roberta-large)  +  LLM-as-judge  +  Entity recall│
│                    → eval_results.json                        │
└──────────────────────────────────────────────────────────────┘
```

### Graph Schema
```
Vertex types : Disease, Drug, Symptom, Treatment, Abstract
Edge types   : treats · causes · contraindicates · mentions

2-hop traversal pattern:
  Seed Entity → MENTIONED_IN → Abstracts → MENTIONED_IN → Co-Entities
```

### Why GraphRAG wins on token count
Basic RAG retrieves **5 full abstract chunks (~600 tokens)** that sound similar to the query.
GraphRAG traverses the knowledge graph and returns a **compact entity list (~50 tokens)** of what actually co-occurs in the literature.

Same answer. 12× less context. That's the 64.2%.

```

├── extract_entities.py     # NER + loads vertices/edges into TigerGraph
```

## 🚀 Setup & Run

### Prerequisites
```
Python 3.10+
TigerGraph Savanna account — free at tgcloud.io (~$60 credits)
Groq API key — free at console.groq.com (14,400 req/day)
HuggingFace token — free at huggingface.co
```

### Install
```bash
git clone https://github.com/YOUR_USERNAME/medgraph-rag.git
cd medgraph-rag
python -m venv venv
venv\Scripts\activate          # Windows
# source venv/bin/activate     # Mac/Linux

pip install pytigergraph requests python-dotenv groq faiss-cpu \
            sentence-transformers bert-score
```

### Environment — create `.env`
```env
GROQ_KEY=your_groq_api_key
TG_HOST=https://your-workspace.tgcloud.io
TG_USER=tigergraph
TG_PASS=your_password
TG_GRAPH=MedGraph
HF_TOKEN=your_hf_token
```

### Run
```bash
# 1. Fetch 13,327 PubMed abstracts (~5.9M tokens)
python fetch_pubmed.py

# 2. Extract entities and load into TigerGraph
python extract_entities.py

# 3. Build FAISS vector index
python build_index.py

# 4. Run full evaluation (all 3 pipelines + BERTScore + Judge)
python evaluate.py
# → writes eval_results.json

# 5. Open dashboard in browser
open dashboard.html
```

## 📈 Evaluation Methodology

| Metric | Method | GraphRAG result |
|---|---|---|
| **Token reduction** | `total_tokens` from Groq usage API, avg over 5 questions | **−64.2%** vs Basic RAG |
| **Answer accuracy** | LLM-as-a-Judge (Groq, `temperature=0`, PASS/FAIL) | **100%** (Bonus ≥ 90% ✅) |
| **Semantic similarity** | BERTScore F1, raw, `roberta-large` | **0.8367** |
| **Entity recall** | Named entity hit rate vs expected entity list | **0.560** vs baseline 0.453 |
| **Cost/1K queries** | tokens × Groq pricing ($0.59/1M in, $0.79/1M out) | **$0.151** vs $0.421 |

### Evaluation questions
All 5 questions are multi-hop co-mention queries — the category where graph traversal has a structural advantage over vector similarity:

1. What drugs are co-mentioned with diabetes in research literature?
2. What symptoms appear in the same studies as metformin?
3. What treatments are researched alongside hypertension?
4. Which diseases are most connected to inflammation in medical research?
5. What drugs and symptoms are associated with cancer treatment research?
